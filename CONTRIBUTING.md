# Contributing to TicketInsight Pro

Thanks for your interest in improving TicketInsight Pro.

This project is aimed at IT operations, support analytics, and ticket intelligence workflows. Contributions that improve reliability, documentation, adapters, analytics, testing, or developer experience are welcome.

## Before You Start

- Check existing issues before opening a new one
- Keep changes focused and easy to review
- Include context about the problem you are solving
- Add or update tests when behavior changes

## Local Development

```bash
git clone https://github.com/alokgarg003/ticketinsight-pro.git
cd ticketinsight-pro
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## Pull Request Guidelines

1. Fork the repository
2. Create a branch with a clear name
3. Make the smallest change that solves the problem well
4. Update documentation if setup, behavior, or APIs change
5. Open a pull request with a short summary, testing notes, and screenshots when relevant

For the longer contributor guide, see [docs/CONTRIBUTING.md](docs/CONTRIBUTING.md).
